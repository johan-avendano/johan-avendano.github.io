Scraper Amazon → MongoDB

Extracción controlada de información de catálogos para construir una base estructurada: identificación (ASIN), título, precio, disponibilidad, marca y recursos de imagen, guardando resultados en MongoDB para procesos posteriores (pricing, publicación y monitoreo).

Python Selenium BeautifulSoup MongoDB Data pipeline

Qué problema resuelve

  • Centraliza información de productos en una base consultable para automatizaciones.
  • Evita trabajo manual de búsqueda/copiar/pegar para cientos o miles de ítems.
  • Permite construir flujos posteriores: pricing, stock, publicación y monitoreo.
  • Reduce inconsistencias al estandarizar campos y normalizar datos.

Qué hace

  • Navega listados/categorías y recorre páginas de resultados (paginación).
  • Entra a la ficha del producto cuando se requiere más detalle.
  • Extrae campos clave: ASIN, título, marca, precio y disponibilidad (según lo visible).
  • Obtiene URLs de imágenes y las guarda como arreglo (para procesamiento posterior).
  • Guarda/actualiza documentos en MongoDB (upsert por ASIN para evitar duplicados).

Flujo resumido

Entrada (categoría/keywords/listado) → Extracción (navegación + parsing) → Normalización → Persistencia (MongoDB) → Salida (colección lista para automatizaciones).

Persistencia MongoDB (documentos por ASIN)
Control Paginar, reintentos, límites y pausas
Calidad Normalización y validación de campos
Uso Base para pricing / publicación / monitoreo
Nota: se evita publicar código sensible. La explicación se enfoca en el diseño del flujo, el manejo de datos y el resultado (estructura en MongoDB).
