Sincronización de precio y stock (Amazon → Mercado Libre)

Automatización para mantener consistencia entre fuente y marketplace: detecta cambios de precio o disponibilidad, aplica reglas de negocio y actualiza publicaciones en Mercado Libre, registrando resultados y evitando desalineación en el catálogo.

Python Selenium Requests / HTTP MongoDB Mercado Libre API

Qué problema resuelve

  • Evita vender con precio desactualizado o sin disponibilidad real.
  • Reduce cancelaciones y reclamos por inconsistencia entre canales.
  • Automatiza actualizaciones repetitivas (en lugar de hacerlo manualmente).
  • Permite monitoreo continuo con trazabilidad de cambios y decisiones.

Qué hace

  • Consulta el estado del producto en la fuente (precio / stock / disponibilidad).
  • Compara contra lo almacenado y detecta cambios relevantes.
  • Aplica reglas (ej. redondeo, márgenes, límites, umbrales, pausa por falta de stock).
  • Actualiza Mercado Libre por API (precio/stock/estado) y valida respuesta.
  • Registra resultados por ítem (OK / error / pendiente) y evidencia de cambios.

Flujo resumido

Fuente (Amazon) → Monitoreo/Comparación → Reglas de negocio → Update (Mercado Libre API) → Persistencia (MongoDB: estados e historial).

Objetivo Consistencia de catálogo
Frecuencia Job programado / ejecución por lotes
Decisión Reglas y umbrales configurables
Trazabilidad Historial de cambios en MongoDB
Nota: no se publican credenciales, tokens ni reglas exactas de negocio. La explicación se enfoca en el diseño del flujo y el resultado (catálogo consistente).
